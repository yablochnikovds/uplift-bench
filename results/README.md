# Benchmark results

Real numbers produced by the benchmark pipeline. Layout:

```
results/
├── benchmark_summary.csv        ← all (dataset, model) rows, with mean ± std across seeds
├── benchmark_summary.md         ← same, formatted for the README
├── per_dataset/                 ← one CSV/MD per dataset (every seed as a row)
│   ├── hillstrom.csv
│   ├── hillstrom.md
│   ├── criteo.csv
│   ├── criteo.md
│   └── …
└── figures/                     ← committed plots referenced from README & docs
    ├── qini_<dataset>.png       ← per-dataset bar chart with bootstrap CIs
    └── heatmap_qini.png         ← model × dataset heatmap (when ≥ 2 datasets)
```

## How the files relate

* The per-dataset CSVs are the primary artefact — each row is one
  `(model, seed)` evaluation with point estimate + 95% BCa bootstrap CI.
* `benchmark_summary.{csv,md}` is the aggregate produced by
  [`scripts/aggregate_results.py`](https://github.com/yablochnikovds/uplift-bench/blob/main/scripts/aggregate_results.py)
  (mean / std across seeds per (dataset, model) cell).
* `figures/` is regenerated by the same aggregate script.

## Reproducing

```bash
uv run python scripts/run_full_benchmark.py \
  --datasets hillstrom synthetic \
  --models s_learner t_learner x_learner r_learner dr_learner class_transformation causal_forest \
  --seeds 42 43 44

uv run python scripts/aggregate_results.py
```

See [`docs/reproducing.md`](https://yablochnikovds.github.io/uplift-bench/reproducing/)
for the full recipe.
